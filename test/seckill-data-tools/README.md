# 秒杀系统数据初始化工具

用于初始化和清理 Redis 中的秒杀测试数据，是运行功能测试和性能测试的前置步骤。

## 功能特性

- **幂等操作**：使用 `SET`/`DEL` 命令，可重复执行不会污染数据
- **批量操作**：使用 Redis Pipeline 批量删除用户记录，提升清理效率
- **支持清理**：测试完成后清理所有测试数据，不影响生产环境

## 文件说明

| 文件 | 说明 |
|------|------|
| `main.go` | 程序入口，解析命令行参数，执行 init/cleanup |
| `go.mod` | 依赖管理（仅依赖 go-redis） |
| `go.sum` | 依赖锁文件 |

## 前置条件

| 服务 | 地址 | 说明 |
|------|------|------|
| Redis | `localhost:6379` | 唯一依赖 |

## 快速开始

### 初始化测试数据

```bash
cd test/seckill-data-tools
go mod tidy
go run . -mode=init
```

### 清理测试数据

```bash
go run . -mode=cleanup
```

### 查看帮助

```bash
go run . -help
# 或不传参数
go run .
```

## 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-mode` | 运行模式：`init` 或 `cleanup` | （必填） |
| `-redis` | Redis 主机地址 | `localhost` |
| `-port` | Redis 端口 | `6379` |

### 使用示例

```bash
# 使用默认配置
go run . -mode=init

# 指定 Redis 地址
go run . -mode=init -redis=192.168.1.100 -port=6380

# 清理数据
go run . -mode=cleanup
```

## 清理范围

清理操作会删除：

1. 所有预置商品的 info/name/stock key
2. 所有功能测试商品的 info/name/stock key
3. 所有性能测试商品的 info/name/stock key
4. 预定义测试用户的购买记录 key
5. 以 `S` 开头的测试订单 key

> 注意：清理**不会**删除非测试用户和非 `S` 开头的订单数据。

## 与 test-e2e 的区别

| 工具 | 目标存储 | 适用场景 |
|------|---------|---------|
| `seckill-data-tools` | Redis only | 配合本地功能/性能测试（gRPC 直连） |
| `test-e2e/seckill_flow` | MySQL + Redis | 完整端到端测试（HTTP API，需启动所有服务） |

`test-e2e` 会在每次运行时自动初始化数据，无需手动调用 `seckill-data-tools`。
